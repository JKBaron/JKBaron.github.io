如何在京东云上部署自己的项目

  • JDK
  • MySql
  • Tomcat
  • 云部署
  • 京东云
  • 远程连接

最近在写项目的时候，一直在本地Tomcat服务器上部署项目，刚开始觉得很爽直到有一天家里停电了，第二天来电当我再次打开电脑启动Tomcat服务器，在我的App上访问服务器的时候，始终没有办法访问，真的是纠结了一整天（烦的我还跟女票吵了一架，现在想起来真不值得）。后来我查看IP地址的时候才恍然大悟，路由器重启，给我分配的IP地址也发生了改变。这叫我苦笑不得呀！那个时候就想买一台虚拟云主机，但是像阿里云这样的云主机一年就得上千RMB，我的天我又不是土豪（不得不承认贵有贵的道理），直到我看到鸿神发了一篇关于京东云的文章彻底拯救了我！这里我有些地方的参考鸿神的文章，后面我又加上了自己的补充，算是给各位开发者一个引导少踩点坑！

申请京东云主机

首先登陆到京东云官网

注册京东云账号，接下来就是选择你的云服务器的配置

套餐一
配置：CPU 1核 内存2GB
时长：2个月
价格：1元

套餐二
配置：CPU 1核 内存2GB
时长：6个月
价格：99元

套餐三
配置：CPU 1核 内存2GB
时长：1年
价格：188元

这个相比阿里云真的是白菜价了，自己想开发个一般的app后台，管理小型网站，小程序后台等等都已经足够了！

99元套餐( 半年 )：

188元套餐（1年）：

这里我选择的是188元套餐（1年），为什么要买一年的，就是京东云是刚新起的云服务，所以为了开拓他的市场，吸引更多的开发者，来完善他的云服务，相比其他云服务厂商低廉的价格无疑是最好的选择。而且活动一结束就要回复几千RMB了。这里给大家讲一个很浅显的道理，第一批用户会在以后都是最先受益的人，这是我在申请360云盘时候得到的体会。

话不多说我们开始选择配置

地域选择离你最近的地域，这样访问比较稳定和快

镜像我们选择CentOS 6.5 64位的镜像,其实我刚开始选择的是Ubuntu 16.04的镜像，但是在安装MySql数据库的时候，云主机会自动给我把数据库删了，导致我在很长的一段时间都是在安装Mysql——删除MySql数据库中度过，后来也联系京东云客服，重置了主机，才发现主机镜像还可以选其他系统的，就果断选择CentOS,也给大家提个醒Ubuntu有这么一个Bug,所以在选择主机方面给大家一个建议。

如果你完成购买，登录jcloud.com，点击控制台：

然后点击云主机：

主要关注的就是这个公网IP了，密码会发在你的手机上，或者重置也可以。

远程连接云主机

使用Xshell远程连接

Xshell下载地址 Xshell安装 一路下一步，这里要注意不要改安装路径，否则会丢失一些文件和失去打开软件的权限，Xshell占不了多少C盘资源。

配置Xshell:

点击 文件->新建

名称-自己根据自己爱好随意定义 协议-ssh

主机-就是你京东云主机的公网IP地址

然后点击确定

选择你刚才创建远程连接，点击连接

点击接受并保存

登录名我们设置为 root

下一步是要输入云服务器的密码，如果你还没有修改过密码，那么就用手机短信或邮箱收到的初始密码，如果已经修改过密码了则用你设置的新密码即可

登陆成功后

我们开始安装Tomcat和JDK

安装JDK和Tomcat

因为Tomcat必须依赖JDK才能运行，所以必须在安装Tomcat之前要提前安装好JDK

我们先切换到home目录下，并在改目录下创建一个名为cloudserver,当然这个文件夹的名字你也可以随便取。进入到该文件夹内。

接下来在此文件夹里下载JDK，值得庆幸的是JDK安装在云主机上都是非常简单的

你没有看错，就是如此的简单。更加强大的是我们平时在Windows平台上下载JDk都必须配置环境变量，在云主机上却不需要配置就可以用。这对于技术小白来说就是“春天又来到”。

远程操作Mysql数据库

首先在你的电脑上下载SQLyog - 64 bit

安装软件也是一路下一步

点击 文件->新连接：

我的sql主机地址-云主机公网IP

密码-就是你安装mysql自己定义的密码

端口-3306

点击连接

我的天！出错了1130，有Bug怎么办？？？？

出现这个错误代码的原因有两个方面：

1.云服务器没有开启3306端口

解决办法：在云主机终端输入

添加一行

-A INPUT -m state –state NEW -m tcp -p tcp –dport 3306 -j ACCEPT

输入命令 ctrl+c

输入命令 ：wq (冒号也要输入)

增加完之后需要重启服务：

sudo service iptables restart

重启之后查看端口号打开状态：使用命令

/etc/init.d/iptables status

2.没有远程操作Mysql数据库的权限

解决办法：

第一句是以权限用户root登录

第二句：选择mysql库

第三句：查看mysql库中的user表的host值（即可进行连接访问的主机/IP名称）

第四句：修改host值（以通配符%的内容增加主机/IP地址），当然也可以直接增加IP地址

第五句：刷新MySQL的系统权限相关表

第六句：再重新查看user表时，有修改。。

重起mysql服务即可完成。
